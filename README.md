# 🏗️ 공사감리용역 실적 계산기

> 전력기술관리법 기준 | 입찰 공고일 기준 3년 실적 자동 산출

---

## 📋 개요

`공사감리용역수행현황확인서` PDF를 업로드하면 입찰 기준에 맞는  
**연면적 합계** 또는 **용역비 합계**를 자동으로 계산하고 PASS/FAIL을 판정합니다.

---

## 📁 파일 구조

```
프로젝트 폴더/
├── app.py            ← 메인 실행 파일 (화면 흐름 제어)
├── parser.py         ← PDF 파싱 모듈

├── requirements.txt  ← 필요 라이브러리 목록

```

## ⚙️ 설치 및 실행

### 1. 라이브러리 설치

VS Code 터미널(`` Ctrl+` ``)에서 아래 명령어 실행:

```bash
pip install streamlit pdfplumber python-dateutil pandas
```

### 2. 프로그램 실행

```bash
streamlit run app.py
```

브라우저가 자동으로 열립니다.  
열리지 않으면 터미널에 표시된 주소(`http://localhost:8501`)를 직접 접속하세요.

### 3. 종료

터미널에서 `Ctrl + C`

---

## 🖥️ 사용 방법

1. 상단 탭에서 계산 모드 선택
   - `📐 연면적 계산` 또는 `💰 용역비 계산`

2. **입찰 공고일** 입력  
   → 자동으로 기준 기간(3년 전 ~ 전날)이 표시됩니다

3. **목표 수치** 입력  
   - 연면적 모드: 목표 연면적 (㎡), 기본값 360,000
   - 용역비 모드: 목표 용역비 (천원)

4. **PDF 파일 업로드**  
   → `공사감리용역수행현황확인서` PDF

5. `계산 시작 →` 버튼 클릭

6. 결과 확인
   - 요약 통계 카드
   - PASS / FAIL 판정 및 달성률
   - 용역별 상세 계산 결과 테이블

## 🧮 계산 공식

### 공통: 가중치(W)
```
W = 인정범위 내 중첩일수 / 전체 감리일수
```
- 시작일과 종료일 **모두 포함**하여 계산

### 연면적 계산 모드
```
환산면적 = 연면적 × 이행비율 × W
```
- PDF 표의 연면적은 이행비율이 **미반영**된 수치 → 직접 곱셈 필요
- 연면적 수치 없는 항목 제외

### 용역비 계산 모드
```
환산용역비 = 용역비 × W
```
- PDF 표의 용역비는 이행비율이 **이미 반영**된 수치 → 추가 곱셈 없음
- 용역비 수치 없는 항목 제외

## 🔍 필터링 규칙

| 조건 | 처리 |
|------|------|
| `비고 참여분야`에 **산업시설** 포함 | ❌ 무조건 제외 |
| 연면적 / 용역비 수치 없음 | ❌ 해당 모드에서 제외 |
| 감리기간이 기준 3년과 **중첩 없음** | ❌ 제외 |
| 위 조건 모두 통과 | ✅ 계산 포함 |

## 📄 지원 PDF 형식

| 컬럼명 | 인식 키워드 |
|--------|------------|
| 분야 | `비고참여분야`, `참여분야`, `분야`, `공사종류` |
| 연면적 | `연면적` |
| 이행비율 | `이행비율`, `이행율` |
| 감리기간 | `감리기간` (시작·종료일 한 셀에 포함) |
| 시작일 | `착공`, `시작`, `개시` |
| 종료일 | `준공`, `종료`, `완료` |
| 용역명 | `용역명`, `공사명`, `용역명칭` |
| 용역비 | `용역비`, `계약금액` |

**감리기간 인식 형식 예시:**
```
2023.12.19
2025.06.30
(560일)
```

## ❓ 자주 묻는 문제

**Q. PDF 업로드 후 "유효한 데이터를 찾지 못했습니다" 오류**  
→ PDF 내 표의 헤더 컬럼명이 위 지원 키워드와 다를 수 있습니다.  
→ 파싱 경고 메시지를 확인해 헤더명을 파악한 후 `parser.py`의 `_build_col_map()` 함수에 키워드를 추가하세요.

**Q. 빨간줄 경고 (Pylance)**  
→ VS Code에서 `Ctrl+Shift+P` → `Python: Select Interpreter`로 올바른 파이썬 선택  
→ 실행 자체는 문제없음

**Q. 이행비율이 없는 행 처리**  
→ 이행비율이 없으면 자동으로 100%로 처리합니다
